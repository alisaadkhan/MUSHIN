## Mushin Creator Search — V3 Discovery Engine Requirements

### **Bug Fixes (Must Remain Fixed)**:

1. **Bug: Engagement "Data Refreshing" State** (FIXED):
   - The `getBenchmarkEngagement()` function provides industry-standard fallback rates per platform and follower tier.
   - ALL results must have `engagement_rate` populated — either from real scraping/enrichment or from the benchmark fallback.
   - The `engagement_source` field MUST be set: `"real_eval"` | `"real_enriched"` | `"benchmark_estimate"`.
   - The UI shows `EST` badge when source is `"benchmark_estimate"`. Never show "Refreshing..." or "Data Refreshing".

2. **Bug: Filter Failure — Follower Range** (FIXED):
   - STRICT ENFORCEMENT at TWO checkpoints:
     a. Pre-merge: `applyFollowerFilter` drops results with null counts when a range is active.
     b. Post-merge: `postMergeFiltered` re-checks after DB profile data merges (follower count may change from DB).
   - Any creator outside the range OR with unknown count is DROPPED when a follower range filter is selected.
   - Fallback tiers NEVER relax the follower filter when the user has explicitly set a range.

### **V3 Feature Requirements**:

**#1 — Platform-Specific Extraction**:
- `_shared/instagram.ts`: `extractInstagramFollowers(text)`, `extractInstagramEngagement(text)`, `extractInstagramProfileImage(html)`
- `_shared/tiktok.ts`: `extractTikTokFollowers(text)`, `extractTikTokEngagement(text)`, `extractTikTokProfileImage(html)`
- `_shared/youtube.ts`: `extractYouTubeSubscribers(text)`, `fetchYouTubeSubscriberCount(username, apiKey)`, `extractYouTubeProfileImage(username)`
- Each extractor matches real Serper snippet patterns for its platform (e.g., "125K Followers", "12.5M Subscribers", "fans", "subs").
- YouTube falls back to Data API (`forHandle` or channel ID) if snippet extraction fails.
- Follower/subscriber counts are MANDATORY — never show "—" in the count field if a range filter is active.

**#2 — Mandatory UI Labels**:
- YouTube: Always display "Subscribers" (not "Followers").
- Instagram/TikTok: Always display "Followers".
- Engagement column: Show real % if available → benchmark % + `EST` badge → `—` if truly no data.

**#3 — 100% Content Precision Filter**:
- `ranking.ts → snippetRelevanceScore()`:
  - `termHitRate` weight: `0.55` (increased from `0.45`)
  - Hard rejection: if `exactHits === 0 && partialHits === 0 && trigramScore < 0.15`, return `0`
  - Keyword density boost: `+0.1` per repeated query term occurrence in snippet
- `niche.ts → inferNiche()`:
  - `THRESHOLD = 5` (from 3) — higher bar for niche confidence
  - `DOMINANCE_GAP = 3` (from 2) — winning niche must dominate by larger margin

**#4 — Clean Display Names**:
- `cleanTitle(raw, username, platform)` strips platform noise: "• Instagram photos and videos", " - YouTube", etc.
- Applied to ALL results (both Serper and DB-first paths).
- Bio truncated at word boundary: `truncateAtWord(text, 200)` — no mid-word cuts.

**#5 — Scoring**:
- `followerPenalty()`: Out-of-range creators receive 0.2× score multiplier. Unknown count → 0.6×.
- `postMergeFiltered` feeds into `sortedResults` (not raw `enrichedResults`).
- `engagement_rate: 0` used as stable sentinel when benchmark value is available — never `null` in response.
