commit 854d9926a3f6c6c6709c0190c38792ecb95dbbf3
Author: alisaadkhan <alisaad75878@gmail.com>
Date:   Wed Apr 1 09:43:12 2026 +0500

    fix(auth): resolve Turnstile 400020 — remove sr-only widget mounting and split per-form captcha state
    
    ISSUE: Turnstile error 400020 blocked all authentication flows.
    
    ROOT CAUSE 1 (Primary):
      The Turnstile widget was mounted inside an sr-only container while waiting
      for a token. sr-only applies width:1px/height:1px/overflow:hidden via CSS,
      which causes Cloudflare's challenge iframe to fail dimension measurement and
      emit error 400020 (invalid widget render context).
    
    FIX: Render the widget in normal document flow at all times. Collapse it
      visually (opacity:0, height:0) only AFTER a token has been successfully
      captured — at that point the widget has already completed its work.
    
    ROOT CAUSE 2 (Secondary):
      A single captchaToken state was shared between the sign-in and sign-up forms.
      Switching tabs would carry a stale token from the other form, causing
      Supabase to reject the mismatched token.
    
    FIX: Each form now has its own independent state object { token, error }
      with matching error handlers and refs, ensuring tokens are always
      consumed by the form that generated them.
    
    ALSO: Added @marsidev/react-turnstile to package.json (was missing).
